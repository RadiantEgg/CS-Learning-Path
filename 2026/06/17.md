# C 项目错误处理设计

##  小项目：错误码 + 错误字符串

适用：

- Lexer
- Mini Grep
- Mini Sort
- 数据结构练习

### 错误码

```c
// 错误码
typedef enum {
    ERR_OK,
    ERR_IO,
    ERR_PARSE,
    ERR_OUT_OF_MEMORY
} ErrorCode;

// 错误字符串
const char *error_string(ErrorCode err)
{
    switch (err) {
    case ERR_OK:
        return "success";
    case ERR_IO:
        return "I/O error";
    case ERR_PARSE:
        return "parse error";
    case ERR_OUT_OF_MEMORY:
        return "out of memory";
    default:
        return "unknown error";
    }
}
// 使用
ErrorCode ret = lexer_next(...);

if (ret != ERR_OK)
    fprintf(stderr, "%s\n", error_string(ret));
```



# C 项目错误处理设计（Error 对象方案）

## 设计目标

解决以下问题：

```c
return -1;
return -2;
return -3;
```

存在的问题：

- 不知道错误码代表什么
- 无法记录具体错误信息
- 无法记录发生位置
- 后期维护困难

---

# 设计思路

将错误信息统一封装到一个 Error 对象中。

函数：

```c
lexer_next(...);
```

负责：

```text
返回：
    成功还是失败

Error对象：
    失败原因
    发生位置
    详细描述
```

---

# Step 1：定义错误码

错误码用于表示错误类别。

```c
typedef enum {
    ERR_OK = 0,

    ERR_IO,                // 文件读写失败
    ERR_PARSE,             // 解析错误
    ERR_INVALID_TOKEN,     // 非法Token
    ERR_OUT_OF_MEMORY      // 内存不足

} ErrorCode;
```

说明：

```text
ERR_OK
    操作成功

ERR_IO
    文件相关错误

ERR_PARSE
    语法解析失败

ERR_INVALID_TOKEN
    非法字符或Token

ERR_OUT_OF_MEMORY
    malloc失败
```

---

# Step 2：定义 Error 对象

用于保存完整错误信息。

```c
typedef struct {

    ErrorCode code;

    int line;
    int column;

    char msg[128];

} Error;
```

字段说明：

| 字段 | 作用 |
|--------|--------|
| code | 错误类型 |
| line | 行号 |
| column | 列号 |
| msg | 错误描述 |

例如：

```c
err.code = ERR_INVALID_TOKEN;
err.line = 5;
err.column = 12;

strcpy(err.msg,
       "unexpected character '$'");
```

表示：

```text
第5行第12列发现非法字符 '$'
```

---

# Step 3：统一错误设置函数

不要在项目中到处写：

```c
err->code = ...
err->line = ...
err->column = ...
strcpy(...)
```

统一封装：

```c
void error_set(
    Error *err,
    ErrorCode code,
    int line,
    int column,
    const char *msg
)
{
    if (err == NULL)
        return;

    err->code = code;
    err->line = line;
    err->column = column;

    snprintf(
        err->msg,
        sizeof(err->msg),
        "%s",
        msg
    );
}
```

作用：

```text
统一生成错误信息

避免重复代码

方便后期修改
```

---

# Step 4：在函数中使用

例如 Lexer：

```c
ErrorCode lexer_next(
    Lexer *lex,
    Token *tok,
    Error *err
)
{
    if (lex == NULL) {

        error_set(
            err,
            ERR_PARSE,
            0,
            0,
            "lexer is NULL"
        );

        return ERR_PARSE;
    }

    if (lex->current == '$') {

        error_set(
            err,
            ERR_INVALID_TOKEN,
            lex->line,
            lex->column,
            "unexpected character '$'"
        );

        return ERR_INVALID_TOKEN;
    }

    return ERR_OK;
}
```

设计原则：

```text
返回值：
    告诉调用者成功还是失败

Error对象：
    告诉调用者具体发生了什么
```

---

# Step 5：调用者处理错误

```c
Error err;

Token tok;

ErrorCode ret;

ret = lexer_next(
    lex,
    &tok,
    &err
);

if (ret != ERR_OK) {

    fprintf(
        stderr,
        "Error[%d] at line %d, col %d: %s\n",
        err.code,
        err.line,
        err.column,
        err.msg
    );

    return 1;
}
```

输出示例：

```text
Error[3] at line 5, col 12:
unexpected character '$'
```

# 优点

## 比 int 更清晰

避免：

```c
return -1;
return -2;
```

改为：

```c
return ERR_PARSE;
```

---

## 错误信息集中管理

统一：

```c
error_set(...)
```

而不是项目里到处赋值。

---

## 支持定位

可记录：

```text
行号
列号
文件名（后续可扩展）
```

---

## 易于扩展

以后可以直接增加：

```c
typedef struct {

    ErrorCode code;

    int line;
    int column;

    const char *file;

    char msg[128];

} Error;
```

不影响整体架构。
